# YOLOv5 模型转换工具 - 图形界面使用指南

## 快速启动

### Windows 系统
双击运行 `run_gui.bat` 或在命令行执行：
```bash
python converter_gui.py
```

### Linux / macOS 系统
```bash
chmod +x run_gui.sh
./run_gui.sh
```

或直接运行：
```bash
python3 converter_gui.py
```

## 界面说明

### 1. 文件选择区域

#### PT模型文件
- 点击「浏览...」按钮选择您的 YOLOv5 `.pt` 模型文件
- 支持所有 YOLOv5 模型（yolov5n, yolov5s, yolov5m, yolov5l, yolov5x）

#### 输出目录
- 选择转换后文件的保存位置
- 如果不选择，默认与输入文件同目录

#### trtexec路径
- 默认值 `trtexec` 表示使用系统 PATH 中的 trtexec
- 如果 trtexec 不在 PATH 中，点击「浏览...」选择 trtexec 可执行文件
- TensorRT 安装路径通常为：`/usr/local/TensorRT/bin/trtexec`

### 2. 模型参数区域

#### 输入名称
- ONNX 模型的输入节点名称
- 默认值：`images`
- YOLOv5 通常使用 `images`

#### 输出名称
- ONNX 模型的输出节点名称
- 默认值：`output`
- YOLOv5 通常使用 `output` 或 `output0`

#### Batch Size
- 批次大小，表示一次推理处理的图片数量
- 默认值：1
- 范围：1-32
- **建议**: 单图推理使用 1，批量推理根据 GPU 内存调整

#### Channels
- 输入图像的通道数
- 默认值：3 (RGB图像)
- 范围：1-4
- **通常保持默认值 3**

#### Width / Height
- 输入图像的宽度和高度
- 默认值：640 × 640
- 范围：32-4096 (必须是32的倍数)
- **常用尺寸**:
  - 640×640 (默认，速度与精度平衡)
  - 1280×1280 (高精度)
  - 416×416 (高速度)

#### 设备
- `cpu`: 使用 CPU 进行转换
- `cuda`: 使用 GPU 进行转换（需要 CUDA 支持）
- **建议**: 如果有 GPU，选择 `cuda` 可加快转换速度

### 3. ONNX 选项区域

#### OPSET版本
- ONNX 操作集版本
- 默认值：12
- 范围：9-17
- **建议**: 保持默认值 12，兼容性最好

#### 简化ONNX模型
- ✓ 启用：优化和简化 ONNX 模型结构，减小文件大小
- **建议**: 保持勾选

#### 动态输入尺寸
- ✓ 启用：支持可变的输入尺寸
- ✗ 禁用：固定输入尺寸，性能更好
- **建议**: 
  - 固定尺寸推理：不勾选（性能更好）
  - 需要多种尺寸：勾选

### 4. TensorRT 选项区域

#### 精度选择

| 精度 | 速度 | 精确度 | GPU内存 | 推荐场景 |
|------|------|--------|---------|----------|
| **FP32** | 1x | 最高 | 最大 | 精度优先场景 |
| **FP16** | 2-3x | 很高 | 中等 | **推荐使用** |
| **INT8** | 4-5x | 良好 | 最小 | 边缘设备 |

- **FP32**: 全精度，无损转换
- **FP16**: 半精度，速度提升2-3倍，精度损失小于1%（推荐）
- **INT8**: 8位整数，需要校准数据，速度最快

#### Workspace (工作空间)
- TensorRT 优化过程中使用的内存大小
- 默认值：4 GB
- 范围：1-16 GB
- **建议**:
  - 小模型 (yolov5n/s): 2-4 GB
  - 中等模型 (yolov5m): 4-6 GB
  - 大模型 (yolov5l/x): 6-8 GB

#### 仅转换为ONNX
- ✓ 勾选：只生成 ONNX 文件，跳过 TensorRT 转换
- 使用场景：
  - 没有安装 TensorRT
  - 只需要 ONNX 模型用于其他框架

### 5. 转换控制区域

#### 🚀 开始转换
- 点击开始转换过程
- 转换过程中按钮变为禁用状态

#### ⏹ 停止
- 中止当前转换过程
- 仅在转换进行中可用

#### 清空日志
- 清除下方的日志显示区域

#### 进度条
- 显示转换进行状态
- 转换过程中会动态显示

### 6. 转换日志区域

实时显示转换过程的详细信息：
- 🔵 蓝色：普通信息
- ✅ 绿色：成功信息
- ⚠️ 橙色：警告信息
- ❌ 红色：错误信息

## 使用步骤

### 基础转换流程

1. **选择文件**
   - 点击「PT模型文件」的「浏览...」按钮
   - 选择您的 `.pt` 模型文件
   - 输出目录会自动设置（也可手动更改）

2. **设置参数**（可选，使用默认值即可）
   - 检查模型参数是否正确
   - 通常保持默认值即可

3. **选择精度**
   - 推荐选择 **FP16** 以获得最佳性能

4. **开始转换**
   - 点击「🚀 开始转换」按钮
   - 观察日志输出
   - 等待转换完成

5. **获取结果**
   - 在输出目录查看生成的文件
   - `.onnx` - ONNX 模型
   - `.engine` - TensorRT 引擎

## 常见问题解决

### 1. 程序无法启动

**问题**: 缺少 tkinter
```bash
# Ubuntu/Debian
sudo apt-get install python3-tk

# CentOS/RHEL
sudo yum install python3-tkinter

# macOS (通常已安装)
brew install python-tk
```

**问题**: 缺少其他依赖
```bash
pip install -r requirements.txt
```

### 2. 找不到 trtexec

**解决方案 1**: 添加到 PATH
```bash
export PATH=$PATH:/usr/local/TensorRT/bin
# 或将此行添加到 ~/.bashrc
```

**解决方案 2**: 在界面中手动选择
- 点击「trtexec路径」的「浏览...」按钮
- 选择 trtexec 可执行文件

### 3. CUDA 相关错误

确保已安装：
- CUDA Toolkit
- cuDNN
- TensorRT

版本需要匹配，例如：
- CUDA 11.8
- cuDNN 8.6
- TensorRT 8.5

### 4. 内存不足

**症状**: 转换过程中崩溃或卡住

**解决方案**:
- 降低 Workspace 大小
- 减小 Batch Size
- 使用更小的输入尺寸
- 使用更小的模型

### 5. 转换速度慢

**优化建议**:
- 设备选择 `cuda` 而不是 `cpu`
- 不勾选「简化ONNX模型」（如果不需要）
- 使用 SSD 而不是 HDD

### 6. 转换失败

**检查清单**:
- ✓ PT 模型文件是否完整
- ✓ PyTorch 版本是否兼容
- ✓ ONNX 版本是否兼容
- ✓ 输出目录是否有写权限
- ✓ 磁盘空间是否充足

## 输出文件说明

转换成功后会生成以下文件：

### yolov5s.onnx
- ONNX 格式模型
- 可用于：
  - ONNX Runtime
  - OpenVINO
  - 其他支持 ONNX 的框架

### yolov5s.engine
- TensorRT 引擎文件
- **注意**: 引擎文件与特定 GPU 绑定
- 不同 GPU 需要重新生成
- 可用于：
  - TensorRT 推理
  - DeepStream
  - Triton Inference Server

## 性能参考

以 YOLOv5s 为例，在 NVIDIA RTX 3080 上：

| 配置 | 转换时间 | 推理速度 |
|------|----------|----------|
| FP32, 640×640 | ~5分钟 | ~8ms |
| FP16, 640×640 | ~3分钟 | ~3ms |
| FP16, 1280×1280 | ~6分钟 | ~10ms |

## 技巧和建议

1. **首次使用**: 先用小模型（yolov5n）测试流程
2. **参数调优**: 从默认参数开始，根据需要逐步调整
3. **日志保存**: 可以复制日志内容保存，方便问题排查
4. **批量转换**: 如需转换多个模型，每次转换完成后再开始下一个
5. **版本管理**: 记录每个转换配置，便于重现
