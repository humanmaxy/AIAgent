## 界面区域详解

### 1. 文件选择区域
位于界面顶部，包含三个文件/目录选择器：

#### PT模型文件
- **功能**: 选择要转换的 YOLOv5 PyTorch 模型文件（.pt格式）
- **操作**: 点击「浏览...」按钮打开文件选择对话框
- **支持**: 所有 YOLOv5 模型版本 (n/s/m/l/x)
- **自动功能**: 选择文件后，如果输出目录为空，会自动设置为模型所在目录

#### 输出目录
- **功能**: 指定转换后文件的保存位置
- **操作**: 点击「浏览...」按钮选择目录
- **自动创建**: 如果目录不存在，程序会自动创建
- **默认**: 与输入文件相同目录

#### trtexec路径
- **功能**: TensorRT 转换工具的位置
- **默认值**: `trtexec` (使用系统PATH中的trtexec)
- **何时修改**: 当 trtexec 不在系统 PATH 中时
- **常见路径**: `/usr/local/TensorRT/bin/trtexec`

### 2. 模型参数区域
配置模型的输入输出规格：

#### 输入名称 / 输出名称
- **用途**: ONNX模型的节点名称
- **YOLOv5默认**: 
  - 输入: `images`
  - 输出: `output` 或 `output0`
- **注意**: 必须与模型实际使用的名称一致

#### Batch Size (批次大小)
- **含义**: 一次推理处理的图片数量
- **范围**: 1-32
- **推荐值**:
  - 实时推理: 1
  - 批量处理: 4-8 (根据GPU内存)
  - 服务器部署: 8-16

#### Channels (通道数)
- **含义**: 图像通道数
- **常用值**:
  - RGB图像: 3
  - 灰度图: 1
  - RGBA图像: 4
- **YOLOv5**: 通常为 3

#### Width / Height (宽度/高度)
- **含义**: 输入图像的尺寸
- **范围**: 32-4096 (建议32的倍数)
- **常用配置**:
  - 高速: 416×416
  - 标准: 640×640 ⭐
  - 高精度: 1280×1280
  - 超高精度: 1920×1920

#### 设备 (Device)
- **cpu**: 使用CPU进行转换（慢但通用）
- **cuda**: 使用GPU进行转换（快但需要CUDA）
- **建议**: 有GPU时选择 cuda

### 3. ONNX 选项区域

#### OPSET版本
- **含义**: ONNX操作集版本
- **范围**: 9-17
- **推荐**: 12 (兼容性最好)
- **其他版本**:
  - 11: PyTorch 1.8-1.9
  - 13: PyTorch 1.10+
  - 14+: 最新特性

#### 简化ONNX模型 ☑
- **功能**: 使用 onnx-simplifier 优化模型
- **效果**: 
  - 减小模型大小 (10-30%)
  - 简化计算图
  - 提升推理速度
- **推荐**: 保持勾选
- **注意**: 需要安装 `onnx-simplifier`

#### 动态输入尺寸 ☐
- **功能**: 允许可变的输入尺寸
- **优点**: 灵活性高，可以处理不同尺寸的图片
- **缺点**: 性能略低于固定尺寸
- **使用场景**:
  - ✓ 需要处理多种尺寸图片
  - ✗ 固定尺寸推理（性能优先）

### 4. TensorRT 选项区域

#### 精度选择

##### FP32 (32位浮点)
- **速度**: 基准 (1x)
- **精确度**: 100%
- **GPU内存**: 最大
- **使用场景**: 精度要求极高的应用

##### FP16 (16位浮点) ⭐ 推荐
- **速度**: 2-3倍于FP32
- **精确度**: 99%+ (mAP损失 < 0.5%)
- **GPU内存**: FP32的50%
- **使用场景**: 大多数生产环境
- **要求**: GPU支持FP16 (大部分现代GPU)

##### INT8 (8位整数)
- **速度**: 4-5倍于FP32
- **精确度**: 95-98% (需要校准)
- **GPU内存**: FP32的25%
- **使用场景**: 边缘设备、实时系统
- **要求**: 需要校准数据集

#### Workspace (工作空间)
- **含义**: TensorRT优化时使用的内存
- **单位**: GB
- **范围**: 1-16 GB
- **推荐配置**:
  - 小模型 (yolov5n/s): 2-4 GB
  - 中模型 (yolov5m): 4-6 GB
  - 大模型 (yolov5l/x): 6-8 GB
- **影响**: 更大的workspace可能获得更好的优化

#### 仅转换为ONNX ☐
- **功能**: 只生成ONNX，跳过TensorRT转换
- **使用场景**:
  - 没有安装TensorRT
  - 需要ONNX用于其他框架
  - 在不同机器上转换TensorRT

### 5. 转换控制区域

#### 🚀 开始转换 按钮
- **功能**: 启动转换流程
- **状态变化**: 
  - 转换中 → 按钮禁用
  - 完成后 → 按钮恢复
- **快捷键**: 可以使用 Enter 键（焦点在按钮上时）

#### ⏹ 停止 按钮
- **功能**: 中止当前转换
- **状态**: 仅在转换进行中启用
- **注意**: 停止后需要重新开始

#### 清空日志 按钮
- **功能**: 清除日志显示区域
- **用途**: 开始新转换前清理界面

#### 进度条
- **类型**: 不确定进度条（动画效果）
- **状态**: 转换时显示动画，空闲时停止
- **用途**: 视觉反馈，表明程序正在工作

### 6. 转换日志区域

#### 日志颜色编码
- 🔵 **蓝色** (info): 常规信息
- ✅ **绿色** (success): 成功消息
- ⚠️ **橙色** (warning): 警告消息
- ❌ **红色** (error): 错误消息

#### 日志内容
- 时间戳: `[HH:MM:SS]`
- 操作步骤
- 文件路径
- 参数配置
- 转换进度
- 错误信息（如有）

#### 日志功能
- **自动滚动**: 新日志自动滚动到底部
- **可选择**: 可以选择和复制日志文本
- **实时更新**: 转换过程实时显示

## 典型使用流程

### 新手流程（全默认）
1. 点击「PT模型文件」→「浏览...」→ 选择 .pt 文件
2. 点击「🚀 开始转换」
3. 等待完成提示
4. 在输出目录查看结果

### 进阶流程（自定义参数）
1. 选择 PT 模型文件
2. 选择输出目录
3. 修改模型参数（如需要）
   - Width: 1280, Height: 1280（高精度）
4. 选择精度: FP16
5. 调整 Workspace: 6 GB
6. 点击「🚀 开始转换」

### 专业流程（完全控制）
1. 选择所有文件和路径
2. 配置精确的模型参数
3. 设置ONNX选项
   - 选择合适的OPSET版本
   - 决定是否使用动态输入
4. 配置TensorRT优化
   - 选择精度
   - 调整workspace
5. 检查日志区域
6. 开始转换
7. 监控日志输出
8. 验证输出文件

## 快捷操作

### 键盘快捷键
- `Enter`: 开始转换（焦点在转换按钮时）
- `Ctrl+C`: 复制选中的日志文本
- `Tab`: 在控件间切换焦点

### 鼠标操作
- **单击按钮**: 执行对应功能
- **双击路径**: 选择整行路径文本
- **滚轮**: 滚动日志内容

## 状态指示

### 转换前
- 转换按钮: ✓ 启用
- 停止按钮: ✗ 禁用
- 进度条: 静止

### 转换中
- 转换按钮: ✗ 禁用
- 停止按钮: ✓ 启用
- 进度条: 动画
- 日志: 实时更新

### 转换完成
- 转换按钮: ✓ 启用
- 停止按钮: ✗ 禁用
- 进度条: 静止
- 弹出: 完成提示框

### 转换失败
- 转换按钮: ✓ 启用
- 停止按钮: ✗ 禁用
- 进度条: 静止
- 日志: 显示错误信息（红色）
- 弹出: 错误提示框

## 最佳实践

1. **首次使用**: 使用默认参数转换一个小模型测试
2. **参数验证**: 开始前检查所有参数是否正确
3. **日志监控**: 转换时观察日志，及时发现问题
4. **保存配置**: 记录成功的配置参数，便于重复使用
5. **批量转换**: 一次转换一个模型，避免资源竞争
6. **结果验证**: 转换后验证输出文件是否正确

- 红色文字: 关键错误
- 橙色文字: 需要注意的警告
- 建议: 复制完整日志用于问题排查
